## Implementation summary
- Added `BodyMemoryBudget`, `BodyMemoryReservation`, and `RequestReceiveResult` in `src/http/io.py`.
- `receive_request_result` now reserves declared body bytes after valid headers and per-request size validation, before reading the remaining body, and rejects aggregate budget exhaustion with `body_memory_budget_exceeded`.
- `ExperimentalHTTPServer` owns a process-wide body memory budget, defaults it to `max_upload_size * max_workers`, exposes it through `--body-memory-budget`, returns a stable JSON `503` for aggregate budget exhaustion, and publishes `body_memory` metrics.
- Reservation release is covered on success, incomplete body, receive exception after reservation, process/request handling exceptions, and the legacy `_receive_request()` byte-returning wrapper.
- README, API, and SECURITY docs now describe the relationship between workers, max request size, and body memory budget.
