## Implementation summary
- `receive_request()` now drops a request as soon as parseable headers declare a body larger than the configured upload cap, avoiding the previous full-body receive path for that case.
- Advanced upload now enforces default caps of 16 MB decoded payload, 64 KB encoded header/chunked-header payload, and 16 KB encoded URL payload; body/base64 encoded data is capped to the encoded size implied by the decoded cap and `--max-size`.
- Over-limit advanced-upload paths return JSON `413` without writing files; valid payloads exactly at explicit test caps still upload successfully.
- The API contract now distinguishes receive-layer oversized request closes from handler/pipeline JSON upload-size errors.
