## Implementation summary
- Added a request admission `BoundedSemaphore` sized to `max_workers`.
- Changed the accept loop to acquire admission before `executor.submit`; overloaded cleartext connections receive a simple `503` response, while TLS connections are closed before handshake.
- Added a short bounded admission wait in the accept loop so capacity freed by quick rejected handshakes can be reused before declaring overload.
- Added a submitted-client wrapper that releases admission on normal close, exceptions, WebSocket lifetime exit, and TLS handshake failure.
- Added canceled-future cleanup for submitted work canceled by shutdown before worker execution.
- Added request admission metrics: `active`, `accepted`, and `rejected`.
- Added unit and live tests for admission saturation, release paths, cancellation, submit failure, metrics, keep-alive, and WebSocket interaction.
